# 🧞 프롬프트지니 × AI 모델 허브 — 통합 확장 프로젝트

## Claude Code 실행용 종합 프롬프트

> 기존 프롬프트지니(promptgenie.kr)에 AI 모델 허브 섹션을 추가하는 프로젝트입니다.
> 기존 Next.js 프로젝트 코드베이스에 새로운 페이지와 컴포넌트를 확장합니다.
> **Phase별로 나눠서** Claude Code에 붙여넣어 실행하세요.

---

# 📋 프로젝트 컨텍스트

```
이 프로젝트는 기존에 운영 중인 프롬프트지니(promptgenie.kr)라는 Next.js 웹사이트에
"AI 모델 허브" 섹션을 새롭게 추가하는 확장 작업이야.

■ 기존 사이트 현황 (promptgenie.kr)
- 기술 스택: Next.js (App Router) + TypeScript
- 주요 기능:
  ① 프롬프트 라이브러리 (/library) — 90개 프롬프트, 카테고리/모델/난이도 필터
  ② 프롬프트 생성기 (/generator) — 4단계 위저드
  ③ 블로그 글쓰기 (/blog-writer)
  ④ 가이드 (/guide)
  ⑤ 블로그 (/blog) — 15개 포스트
  ⑥ 내 프롬프트 (/profile)
- 네비게이션: 상단 헤더에 메뉴 (라이브러리, 프롬프트 생성기, 블로그 글쓰기, 가이드, 블로그, 내 프롬프트)
- 디자인: 다크 테마 기반, 보라/블루 그라데이션 액센트, 깔끔한 카드형 UI
- 검색: ⌘K 커맨드 팔레트
- 다국어: /ko 경로 기반 한국어

■ 확장 목표
기존 사이트의 디자인 시스템, 컴포넌트, 레이아웃을 최대한 재활용하면서
"AI 모델 허브"를 새로운 메가 섹션으로 추가해.
기존 기능은 절대 건드리지 말고, 새 페이지/컴포넌트만 추가하는 방식으로 진행해.

■ 핵심 원칙
1. 기존 코드 수정 최소화 — 네비게이션에 메뉴 추가와 홈페이지 위젯 추가만 수정
2. 기존 디자인 시스템 준수 — 동일한 색상, 폰트, 카드 스타일, 간격 사용
3. 크로스 링킹 — 프롬프트와 AI 모델 사이의 상호 연결
4. 모든 UI 텍스트는 한국어
```

# Phase 1 — 프로젝트 구조 파악 & 확장 계획

```
먼저 기존 프로젝트 구조를 파악하고 확장 계획을 세워줘.

■ Step 1: 기존 구조 분석
아래 내용을 확인해서 보고해줘:
- src/ 또는 app/ 디렉토리 구조
- tailwind.config (색상, 폰트 등 디자인 토큰)
- 공통 컴포넌트 목록 (components/ 폴더)
- 레이아웃 파일 (layout.tsx)
- 기존 네비게이션 컴포넌트
- package.json (의존성 목록)
- 기존 데이터 관리 방식 (DB? JSON? CMS?)

■ Step 2: 확장할 파일/폴더 구조 계획
기존 구조를 바탕으로, 아래와 같은 새 경로들을 추가할 거야:

신규 페이지:
  /models                → AI 모델 허브 메인 (대시보드)
  /models/[id]           → AI 모델 상세 페이지
  /models/compare        → 모델 비교
  /models/benchmarks     → 벤치마크 리더보드
  /models/pricing        → 가격 계산기
  /models/timeline       → 출시 타임라인
  /news                  → AI 뉴스 & 트렌드
  /glossary              → AI 용어사전

신규 컴포넌트 폴더:
  components/models/     → 모델 허브 전용 컴포넌트
  components/dashboard/  → 대시보드 위젯
  components/charts/     → 차트 컴포넌트

신규 데이터:
  data/models.json       → AI 모델 데이터
  data/providers.json    → AI 기업 정보
  data/benchmarks.json   → 벤치마크 정보
  data/ai-news.json      → AI 뉴스
  data/glossary.json     → AI 용어사전
  data/timeline.json     → 출시 타임라인

■ Step 3: 필요한 패키지 설치
기존에 없다면 아래 패키지를 추가 설치해줘:
- recharts (차트/시각화)
- framer-motion (애니메이션) — 이미 있을 수 있음
- 기타 필요한 것이 있으면 판단해서 추가

기존 코드를 분석한 뒤, 확장 계획을 정리해서 보여주고,
확인받은 후 다음 Phase로 넘어갈게.
```

# Phase 3 — 네비게이션 확장 & AI 모델 허브 메인 대시보드

```
이 Phase에서는 두 가지를 해줘:
(A) 기존 네비게이션에 AI 모델 허브 메뉴 추가
(B) /models 메인 대시보드 페이지 제작

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ (A) 네비게이션 확장
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

기존 헤더 네비게이션을 찾아서, 기존 메뉴는 그대로 두고
"AI 모델" 메뉴 항목을 추가해줘.

방식 1 (메가 메뉴): "AI 모델" 호버/클릭 시 드롭다운
  - 모델 허브 (대시보드)  → /models
  - 모델 비교             → /models/compare
  - 벤치마크              → /models/benchmarks
  - 가격 계산기           → /models/pricing
  - 출시 타임라인         → /models/timeline
  - AI 뉴스              → /news
  - AI 용어사전           → /glossary

방식 2 (단일 링크): "AI 모델" → /models 로 이동, 나머지는 서브 네비게이션
→ 기존 네비게이션 스타일에 더 자연스러운 방식을 선택해.

또한 ⌘K 커맨드 팔레트에도 새 페이지들을 검색 가능하도록 추가해줘 (가능하다면).

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

# 🔧 실행 가이드

| Phase | 내용 | 예상 소요 | 핵심 포인트 |
|-------|------|----------|------------|
| 1 | 구조 파악 & 계획 | 5~10분 | 기존 코드 분석이 핵심 |
| 2 | 데이터 구축 | 20~30분 | 정확한 데이터 = 성공의 절반 |
| 3 | 네비 확장 + 대시보드 | 25~35분 | 가장 큰 작업, 핵심 페이지 |
| 4 | 모델 목록 + 상세 | 20~30분 | 필터/탭/크로스링킹 |
| 5 | 비교 + 벤치마크 | 20~25분 | 차트 중심 |
| 6 | 가격 + 타임라인 | 20~25분 | 인터랙티브 위젯 |
| 7 | 뉴스 + 용어 + 홈확장 | 20~25분 | 크로스 링킹 마무리 |
| 8 | 폴리싱 | 15~20분 | 반응형, 애니메이션, SEO |

### 💡 Claude Code 실행 팁

1. **Phase 1을 반드시 먼저** — 기존 코드 구조를 파악해야 나머지가 수월합니다.
2. **Phase 2 데이터를 꼼꼼히 검토** — 벤치마크 점수 등 잘못된 데이터가 있으면 이후 모든 차트에 영향.
3. **각 Phase 끝날 때마다 `npm run dev`** 로 확인 후 다음 Phase로.
4. **기존 코드는 건드리지 마** 라는 원칙을 반복 강조해주세요.
5. **디자인 일관성** — "기존 사이트의 카드/색상/폰트를 그대로 써" 라고 자주 리마인드.
6. 에러 발생 시 에러 메시지를 그대로 붙여넣으면 빠르게 해결됩니다.
7. Phase 3이 가장 크므로, 대시보드 위젯을 3~4개씩 나눠서 진행해도 좋습니다.
